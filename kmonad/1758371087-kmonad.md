# kmonad
- kmonad & kanata are key remapping software that use system kernel extensions under the hood
  - this uses the karabiner virtual HID device drive
- there are some config files that you need
- rather than run manual commands every time at startup, we use plist files and the launchctl
system to automate this
- [kmonad macos docs](https://github.com/kmonad/kmonad/blob/a38004ff4df4beda648ec5b1374173d3192d61ae/doc/installation.md?plain=1#L338)
  - There are docs on how to install the Karabiner dext embedded here.
  - The binary is part of the Kmonad source tree -- or you can install it from the actual Karabinier project
  - There's also a plist file provided [here](https://github.com/pqrs-org/Karabiner-DriverKit-VirtualHIDDevice/blob/v5.0.0/files/LaunchDaemons/org.pqrs.service.daemon.Karabiner-VirtualHIDDevice-Daemon.plist)
    - This file ensures that the Daemon is started at startup: `/Library/LaunchDaemons/com.example.karabiner-vhiddaemon.plist`
    - This File Is Extraneous: `/Library/LaunchDaemons/com.example.karabiner-vhidmanager.plist`

*commands*
```
# Karabiner setup

# Starting & Stopping the launchctl automation
sudo launchctl unload /Library/LaunchDaemons/com.example.kanata.plist
sudo launchctl load /Library/LaunchDaemons/com.example.kanata.plist
sudo launchctl bootstrap system /Library/LaunchDaemons/com.example.kanata.plist
```

# Making Changes
- You can make changes to the config file `~/.config/kmonad.kbd`
- Then load & unload the plist

# Caveats
- Needs special permissions & you need to update System Settings
    - `System Settings` > `Privacy & Security` > `Input Monitoring` > `{Alacritty,kmonad,kanata}` binaries
- Check to see if it's running:
    - `Keyboard` > `Keyboard Shortcuts` > `Modifier Keys` > `Select Keyboard`

# DEBUGGING
- If you can't figure out what's wrong. Try to run `kmonad` manually & see if that shows some errors
- `kmonad` will fail silently when using the automation and you won't be able to tell what's wrong
